Wordpress MVC gives you a stable development environment that allows you to organize your code outside of the templating engine.

It vows to keep Wordpress unmodified while doing so.

Be up and running in 4 simple steps:

Step 1Install

Wordpress MVC can be install using a Yeoman generator.

$ mkdir my-website && cd my-website
$ yo wordpress-mvc

This will generate a set of tools inside a solid and secure directory structure, containing with a preconfigured VM that is dedicated only for this one project.

Step 2Create a page

Using Wordpress' backend, create a page that has a permalink of /songs/my-song-list/ and insert a shortcode named list_songs in the post's body.

<p>One can customize the page's content.</p>
[list_songs]
<p>This would be under the generated content of the shortcode.</p>

Step 3Declare a route

Set up a route pointing to that page:

array('GET',        '/songs/my-song-list/',           'SongsController#index')
array('GET',        array('/songs/my-song-list/?[*:slug]?' => 'index.php?pagename=songs/my-song-list'), 'SchoolsController#view')

Step 4Code

All requests to /songs/my-song-list/ will be processed by a controller class similar to the following one. They will be ran before compiling the usual Wordpress templates.

<?php
namespace Mywebsite\Controllers;

use MVC\Controller;
use Mywebsite\Models\Song;

class SongsController extends Controller {

    public $shortcodes = array("list_songs" => 'getSongsListingShortcode');

    public function index()
    {

    }

    public function view($slug = null)
    {
        if (!is_null($slug)) {
            $this->set("song", Song::findBySlug($slug));
        }
    }

    public function getSongsListingShortcode()
    {
        $list = Song::query()->listing("ID", "post_title");
        return "<p>" . implode(", ", $list) . "</p>";
    }
}
