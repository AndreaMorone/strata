Gain access to the MVC programming pattern in Wordpress.

Wordpress is a fantastic blog engine and is more and more used as a full fledged CMS. When building more complex websites or applications, one may find that you can't quite cleanly separate the business logic from the templating engine.

Be up and running in 4 simple steps:

Step 1Install

Wordpress MVC is a Composer dependency.

"require": {
    "francoisfaubert/wordpress-mvc": "dev-master"
}

Step 2Create a page

Use the Wordpress engine just like you have been for years.

Here the page has a permalink of /schools/a-random-title/.

Step 3Declare route

Set up a route pointing to that page:

array('POST|GET',        '/schools/[*:slug]/',           'SchoolsController#view')

Step 4Code

And then all requests to "/schools/a-random-title/" will go though a controller class like this one before compiling the templates.

namespace Mywebsite\Controllers;

use MVC\Controller;
use Mywebsite\Model\School;
use Mywebsite\Model\Logger;

class SchoolController extends Controller {

    public $shortcodes = array("list_schools" => 'getSchoolListingShortcode');

    public function before()
    {
        Logger::write(time());
    }

    public function view($schoolId = null)
    {
        if (isset($schoolId)) {
            $this->set("school", School::getById((int)$schoolId));
        }
    }

    public function getSchoolListingShortcode()
    {
        $list = School::listing();
        return "<p>" . implode(", ", $list) . "</p>";
    }
}
