Gain access to the MVC programming pattern in Wordpress.

Wordpress is a fantastic blog engine and is more and more used as a full fledged CMS. When building more complex websites or applications, one may find that you can't quite cleanly separate the business logic from the templating engine.

Be up and running in 4 simple steps:

Step 1Install

Wordpress MVC is a Composer dependency.

"require": {
    "francoisfaubert/wordpress-mvc": "dev-master"
}

Step 2Create a page

Use the Wordpress engine just like you have been for years.

Create a page that has a permalink of /songs/a-random-title/ and insert a shortcode named list_songs in the post's body.

Step 3Declare route

Set up a route pointing to that page:

array('POST|GET',        '/songs/[*:slug]/',           'SongsController#view')

Step 4Code

And then all requests to "/songs/a-random-title/" will go though a controller class like this one before compiling the templates.

<?php
namespace Mywebsite\Controllers;

use MVC\Controller;
use Mywebsite\Model\Songs;
use Mywebsite\Model\Logger;

class SongsController extends Controller {

    public $shortcodes = array("list_songs" => 'getSongsListingShortcode');

    public function before()
    {
        Logger::write(time());
    }

    public function view($songId = null)
    {
        if (isset($songId)) {
            $this->set("song", Songs::getById((int)$songId));
        }
    }

    public function getSongsListingShortcode()
    {
        $list = Songs::listing();
        return "<p>" . implode(", ", $list) . "</p>";
    }
}
