    • includeUtils() — Includes the debugger classes.

    • saveConfigurationFileSettings() — Saves an array of configuration attribute to the application.
    • saveCurrentPID() — Save the latest PHP process ID as a temp file in case an infinite loop or any unexpected error breaks the server and prevents Strata from closing it automatically.

    public

    • addProjectNamespaces() — Adds the current project namespace to the project's class loader.
    • getLoader() — Returns a class loader to the application
    • getLogger() — Returns a reference to the Logger. When no $name is provided, attempts to return the most plausible one.
    • getMiddlewares() — Returns the list of declared middlewares in the application.
    • includeGettextFixture() — Loads up a gettext fixture against which the tests will test.
    • includeWordpressFixture() — Loads up a Wordpress fixture on which the tests will register filters and actions.
    • init() — Prepares the Strata object for running.
    • loadConfiguration() — Loads the configuration file and assigns it to the running app.
    • log() — Sends a message to the active logger
    • registerErrorHandler()
    • run() — Kickstarts the Router and preload the custom post types associated to the project.
    • setDefaultNamespace() — Sets the default namespaces Strata knows about in the project's Composer's ClassLoader.
    • setLoader() — Assigns a class loader to the application
    • takeOverWPCLIArgs() — ~/src/Scripts/runner.php and WP-CLI command line scripts do not have the same amount of parameters. This takes over WP-CLI's order of arguments and registers them back for Strata.
